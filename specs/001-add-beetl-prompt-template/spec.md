**功能分支**: `001-add-beetl-prompt-template`  
**创建日期**: 2025-11-27  
**状态**: 草稿  
**输入**: 用户描述: "添加Beetl模板引擎库, 添加基本配置, 使其能用; 根据prompt_template表定义, 编写增删改查接口"

## 用户场景与测试 *（必填）*

### 用户故事 1 - Beetl 模板引擎基础配置 (优先级: P1)

作为系统管理员，我希望系统能够集成 Beetl 模板引擎并完成基础配置，以便后续能够使用 Beetl 进行模板渲染。

**设定此优先级的理由**: Beetl 模板引擎是整个 Prompt 模板功能的基础依赖，没有它后续的模板渲染功能无法实现。

**独立测试**: 可通过编写简单的 Beetl 模板字符串进行渲染测试，验证模板引擎是否正确初始化和配置。

**验收场景**:

1. **假如** 系统已启动，**当** 调用 Beetl 模板渲染服务并传入模板字符串和参数 时，**那么** 系统返回正确渲染后的结果字符串
2. **假如** 模板中包含变量占位符，**当** 传入对应的参数值 时，**那么** 占位符被正确替换为参数值
3. **假如** 模板语法错误，**当** 尝试渲染 时，**那么** 系统返回友好的错误信息

---

### 用户故事 2 - 创建 Prompt 模板 (优先级: P1)

作为运营人员，我希望能够创建新的 Prompt 模板，以便为不同的 AI 角色配置专属的对话模板。

**设定此优先级的理由**: 创建模板是模板管理的核心功能，是其他操作（查询、更新、删除）的前提。

**独立测试**: 可通过调用创建接口，传入模板内容和相关参数，验证模板是否成功保存到数据库。

**验收场景**:

1. **假如** 用户填写了必填字段（角色ID、模板内容），**当** 提交创建请求 时，**那么** 系统成功创建模板并返回模板ID
2. **假如** 用户未填写必填字段，**当** 提交创建请求 时，**那么** 系统返回字段校验错误信息
3. **假如** 同一角色已存在模板，**当** 创建新版本模板 时，**那么** 系统自动递增版本号并将新模板标记为最新版本

---

### 用户故事 3 - 查询 Prompt 模板列表 (优先级: P1)

作为运营人员，我希望能够按条件查询 Prompt 模板列表，以便快速找到需要管理的模板。

**设定此优先级的理由**: 查询功能是日常模板管理的高频操作，支撑运营人员的日常工作。

**独立测试**: 可通过调用查询接口，传入不同的筛选条件，验证返回结果是否符合预期。

**验收场景**:

1. **假如** 数据库中存在多个模板，**当** 按角色ID查询 时，**那么** 系统返回该角色的所有模板列表
2. **假如** 用户指定分页参数，**当** 查询模板列表 时，**那么** 系统按分页参数返回对应的数据
3. **假如** 用户按状态筛选，**当** 查询模板列表 时，**那么** 系统仅返回匹配状态的模板

---

### 用户故事 4 - 查询单个 Prompt 模板详情 (优先级: P2)

作为运营人员，我希望能够查看单个 Prompt 模板的完整详情，以便了解模板的具体配置。

**设定此优先级的理由**: 详情查询是编辑和调试模板的前置步骤。

**独立测试**: 可通过调用详情接口，传入模板ID，验证返回的模板详情是否完整。

**验收场景**:

1. **假如** 模板ID存在，**当** 查询模板详情 时，**那么** 系统返回该模板的所有字段信息
2. **假如** 模板ID不存在，**当** 查询模板详情 时，**那么** 系统返回"模板不存在"的错误提示

---

### 用户故事 5 - 更新 Prompt 模板 (优先级: P2)

作为运营人员，我希望能够更新已有的 Prompt 模板信息，以便修正错误或优化模板内容。

**设定此优先级的理由**: 模板内容需要持续迭代优化，更新功能支撑模板的生命周期管理。

**独立测试**: 可通过调用更新接口，修改模板的某个字段，验证数据库中的值是否正确更新。

**验收场景**:

1. **假如** 模板ID存在，**当** 提交更新请求 时，**那么** 系统成功更新模板信息
2. **假如** 更新的字段不符合校验规则，**当** 提交更新请求 时，**那么** 系统返回校验错误信息
3. **假如** 模板ID不存在，**当** 提交更新请求 时，**那么** 系统返回"模板不存在"的错误提示

---

### 用户故事 6 - 删除 Prompt 模板 (优先级: P2)

作为运营人员，我希望能够删除不再需要的 Prompt 模板，以便保持模板库的整洁。

**设定此优先级的理由**: 删除功能完善模板的生命周期管理，避免无效模板干扰正常使用。

**独立测试**: 可通过调用删除接口，传入模板ID，验证模板是否被正确标记为已删除。

**验收场景**:

1. **假如** 模板ID存在，**当** 提交删除请求 时，**那么** 系统将模板标记为逻辑删除状态
2. **假如** 模板ID不存在，**当** 提交删除请求 时，**那么** 系统返回"模板不存在"的错误提示
3. **假如** 模板已被删除，**当** 查询模板列表 时，**那么** 已删除的模板不会出现在结果中

---

### 用户故事 7 - 模板渲染预览 (优先级: P3)

作为运营人员，我希望能够预览模板的渲染效果，以便在发布前确认模板内容正确。

**设定此优先级的理由**: 预览功能提升模板配置的准确性，减少线上错误。

**独立测试**: 可通过调用预览接口，传入模板ID和测试参数，验证渲染结果是否符合预期。

**验收场景**:

1. **假如** 模板ID存在，**当** 传入测试参数并请求预览 时，**那么** 系统返回渲染后的模板内容
2. **假如** 模板参数结构与传入参数不匹配，**当** 请求预览 时，**那么** 系统返回参数缺失或类型错误的提示

---

### 边缘场景

- 当模板内容为空字符串时，系统如何处理？→ 系统拒绝创建/更新，返回"模板内容不能为空"
- 当传入的渲染参数包含特殊字符或恶意脚本时，系统如何处理？→ Beetl 模板引擎默认不执行脚本，参数作为纯文本处理
- 当同时更新同一模板时，系统如何处理并发冲突？→ 采用乐观锁机制，后提交的请求提示"数据已被修改，请刷新后重试"
- 当模板版本号达到上限时，系统如何处理？→ INT UNSIGNED 最大值约 42 亿，实际不会达到，无需特殊处理

## 需求 *（必填）*

### 功能性需求

#### Beetl 模板引擎集成

- **FR-001**: 系统**必须**集成 Beetl 模板引擎作为模板渲染工具
- **FR-002**: 系统**必须**提供模板渲染服务，支持传入模板字符串和参数 Map 进行渲染
- **FR-003**: 系统**必须**在模板语法错误时返回清晰的错误信息，包含错误位置和原因

#### Prompt 模板 - 创建

- **FR-004**: 系统**必须**支持创建 Prompt 模板，必填字段包括：角色ID (char_id)、模板内容 (content)
- **FR-005**: 系统**必须**在创建模板时自动设置版本号，首次创建为 1，同角色后续创建自动递增
- **FR-006**: 系统**必须**在创建新版本时自动将该版本标记为最新版本 (is_latest=1)，并将该角色的其他版本标记为非最新 (is_latest=0)
- **FR-007**: 系统**必须**在创建时自动记录创建人和创建时间

#### Prompt 模板 - 查询

- **FR-008**: 系统**必须**支持分页查询 Prompt 模板列表
- **FR-009**: 系统**必须**支持按角色ID (char_id) 筛选模板
- **FR-010**: 系统**必须**支持按状态 (status) 筛选模板
- **FR-011**: 系统**必须**支持按是否最新版本 (is_latest) 筛选模板
- **FR-012**: 系统**必须**支持按ID查询单个模板的完整详情
- **FR-013**: 系统**必须**在查询时自动排除已逻辑删除的记录 (is_deleted=1)

#### Prompt 模板 - 更新

- **FR-014**: 系统**必须**支持按ID更新模板信息
- **FR-015**: 系统**必须**在更新时自动记录更新人和更新时间
- **FR-016**: 系统**必须**支持更新模板状态（草稿/启用/停用）
- **FR-017**: 系统**必须**支持更新模板的灰度配置（灰度策略、灰度比例、灰度用户白名单）
- **FR-018**: 系统**必须**支持将模板标记为稳定版本 (is_stable=1)

#### Prompt 模板 - 删除

- **FR-019**: 系统**必须**支持按ID逻辑删除模板（设置 is_deleted=1）
- **FR-020**: 系统**不得**提供物理删除模板的功能

#### Prompt 模板 - 渲染预览

- **FR-021**: 系统**必须**支持传入模板ID和参数进行渲染预览
- **FR-022**: 系统**必须**在预览时校验参数是否符合模板定义的参数结构 (param_schema)

### 关键实体

- **Prompt 模板 (prompt_template)**: 存储 AI 角色的对话模板，包含模板内容（Beetl 语法）、适用模型、语言、版本管理信息、灰度发布配置等。每个模板归属于一个角色 (char_id)，支持多版本管理，可配置灰度发布策略。

## 假设

1. **权限控制**: 假设当前阶段不需要细粒度的权限控制，所有接口对已认证用户开放
2. **模板语法**: 假设使用 Beetl 默认的定界符配置（`${}`）
3. **参数结构**: 假设 param_schema 字段使用 JSON Schema 格式描述参数结构
4. **分页默认值**: 假设分页查询默认每页 10 条，最大每页 100 条
5. **创建人/更新人**: 假设从当前登录用户上下文中获取
