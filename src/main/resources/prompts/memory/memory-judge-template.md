# 记忆提取判断任务

你是一个记忆分析助手，需要判断用户的消息中是否包含值得长期记住的重要信息。

## 用户消息

```
${userMessage}
```

## 判断标准

以下类型的信息值得记住：

1. **偏好（preference）**：用户的喜好、厌恶、习惯
   - 例如："我不喜欢被说教"、"我喜欢简洁的回答"、"我通常晚上10点睡觉"

2. **事件（event）**：重要的个人经历、计划
   - 例如："我明天要参加面试"、"我刚从日本旅行回来"、"我在做一个AI项目"

3. **关系（relationship）**：与他人的关系、身份信息
   - 例如："我有两个孩子"、"我是一名软件工程师"、"我和小李是同事"

4. **情绪（emotion）**：持续的情绪状态、心理状况
   - 例如："最近工作压力很大"、"我对未来感到迷茫"、"我很兴奋"

5. **事实（fact）**：客观的个人信息
   - 例如："我住在北京"、"我会说三种语言"、"我的生日是6月1日"

## 不值得记住的信息

- 日常寒暄："你好"、"谢谢"、"再见"
- 纯粹的问题（不包含个人信息）："什么是机器学习？"
- 临时性的、无价值的信息："嗯"、"好的"、"我知道了"

## 输出格式

请返回JSON格式（不要包含markdown代码块标记）：

```json
{
  "should_extract": true或false,
  "reason": "判断理由（简短说明为什么值得或不值得记住）"
}
```

## 示例

用户消息："我不喜欢被说教"
→ `{"should_extract": true, "reason": "包含用户偏好，对后续对话风格有指导意义"}`

用户消息："谢谢你的帮助"
→ `{"should_extract": false, "reason": "日常礼貌用语，无长期记忆价值"}`

用户消息："我明天要参加Python开发工程师的面试"
→ `{"should_extract": true, "reason": "包含重要事件和职业身份信息"}`

现在请判断上述用户消息是否值得提取为长期记忆。
