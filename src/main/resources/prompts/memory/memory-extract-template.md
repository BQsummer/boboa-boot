 记忆提取任务

你是一个专业的记忆提取助手，需要从用户消息中提取值得长期记住的信息，并分类整理。

## 用户消息

```
${userMessage}
```

## 提取要求

从上述消息中提取**所有**值得记住的信息点，每个信息点需要：

1. **text**: 记忆的文本内容（第一人称表述，简洁明确）
   - 好的示例："我不喜欢被说教"、"我明天有Python面试"、"我住在北京"
   - 不好的示例："用户说他不喜欢..."（不要用第三人称）

2. **type**: 记忆类型，必须是以下之一：
   - `preference`: 偏好、习惯
   - `event`: 事件、计划
   - `relationship`: 关系、身份
   - `emotion`: 情绪、心理状态
   - `fact`: 客观事实

3. **importance**: 重要性评分（0.0-1.0的浮点数）
   - 0.9-1.0: 极其重要，核心身份信息或强烈偏好
   - 0.7-0.9: 重要，明确的偏好或重要事件
   - 0.5-0.7: 中等重要，一般性信息
   - 0.3-0.5: 较低重要，临时性信息
   - 0.0-0.3: 不建议使用（应该在判断阶段过滤）

4. **reason**: 提取原因（简短说明为什么这个信息重要）

## 输出格式

请返回JSON格式（不要包含markdown代码块标记）：

```json
{
  "memories": [
    {
      "text": "第一人称的记忆内容",
      "type": "类型",
      "importance": 0.8,
      "reason": "提取理由"
    }
  ]
}
```

## 示例

### 示例1
用户消息："我不喜欢被说教，希望你能用平等的语气和我交流"

输出：
```json
{
  "memories": [
    {
      "text": "我不喜欢被说教",
      "type": "preference",
      "importance": 0.85,
      "reason": "明确的对话风格偏好，对后续交流至关重要"
    },
    {
      "text": "我希望用平等的语气交流",
      "type": "preference",
      "importance": 0.8,
      "reason": "沟通方式偏好，影响对话体验"
    }
  ]
}
```
